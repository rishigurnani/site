Learning

Discover my brief takes on Density Functional Theory and Machine Learning-along with curated lectures and tutorials—to empower emerging researchers.

My work leverages computational tools to study porous materials. These computational tools include Density Functional Theory (DFT) and Machine Learning (ML). At a high level, DFT allows us to set up several interesting quantum-mechanical equations. By solving them, we can learn about the electronic properties of a set of atoms to an accuracy which matches physical experiment. To learn more about DFT view Rasoul Khaledialidusti‘s four-part series. A good compliment to this series is a set of lectures given by my PhD advisor, Dr. Ramprasad, on Electronic Structure Theory.

In addition to the electronic properties of a material, we also sometimes wish to understand its phononic properties. Phonons are just the coordinated movements of atoms in a material. The phononic properties of a material are uncovered by obtaining its Dynamical Matrix. To learn more about the Dynamical Matrix, I recommend checking out this easy-to-digest paper by M.T. Dove. In a nutshell, the Dynamical Matrix tells us how displacing one atom in a material changes the force on the other constituent atoms. It turns out this displacement-force relationship, and thus the phononic properties of a material, can also be computed using DFT methods. G J Ackland wrote a nice review of these methods which you can find here.

Although incredibly powerful, DFT is slow. That is, it takes a long time to compute properties for systems containing many atoms. ML offers a speedy alternative. An incredible introduction to ML, recorded by Andrew Ng, can be found here. At a high level, ML is just a way of approximating maps. A map is a fancy mathematical term that really just means “a set of arrows which show us the relationships between all possible inputs and their corresponding outputs”. For example, the map below details a possible relationship between numbers and letters.

There are many flavors of ML algorithms and each has a unique way of using data to approximate the underlying map. Probably the most popular of these algorithms is the neural network. With five minutes and a little bit of Python knowledge you can create your very own neural network using the Keras API. Here’s a good tutorial for that (be sure to first look at the link inside which shows you how to set up your “environment”). Although Keras is quick and easy, there are much more powerful neural networks that can be built (such as a Variational Autoencoder) with a little bit more knowledge. There’s a great set of free Stanford lectures which shows you how to do this in Tensorflow (basic Python knowledge needed). Another potentially helpful resource is my lecture on debugging neural networks.

Another ML algorithm I use frequently is Gaussian Process Regression (GPR). A gentle introduction to GPR can be found here (basic understanding of probability notation and concepts is helpful).
